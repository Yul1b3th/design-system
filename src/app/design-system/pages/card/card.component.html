Card

El componente de tarjeta (ds-card) se utiliza para mostrar contenido en un formato de tarjeta. Este componente es altamente personalizable y soporta múltiples configuraciones para adaptarse a diferentes contextos de uso.

Uso

Ejemplo de implementación en HTML:


      <ds-card
        [imageUrl]="'https://ezhub.vercel.app/assets/img/bedrooms/1.jpg'"
        [title]="'Card Title'"
        [body]="'This is a card body text.'"
        [buttonText]="'Learn More'"
        [buttonUrl]="'#'"
      ></ds-card>

Ejemplo

Accesibilidad

El componente de tarjeta incluye soporte para accesibilidad mediante el uso de atributos aria y roles adecuados.

Atributos ARIA

Los siguientes atributos ARIA se utilizan en el componente de tarjeta:

  • aria-label: Proporciona una etiqueta accesible para elementos interactivos como botones.
  • aria-labelledby: Asocia un elemento con su etiqueta. Utilizado para asociar la tarjeta con su título.
  • aria-describedby: Proporciona una descripción adicional para un elemento. Utilizado para asociar la tarjeta con su cuerpo.

Roles

Los siguientes roles se utilizan en el componente de tarjeta:

  • role="region": Indica que el contenedor principal de la tarjeta es una región del contenido.
  • role="img": Indica que un elemento es una imagen.

Ejemplo

Aquí hay un ejemplo de cómo se pueden utilizar estos atributos y roles en el HTML: 


  <ds-card
    [imageUrl]="'https://ezhub.vercel.app/assets/img/bedrooms/1.jpg'"
    [title]="'Card Title'"
    [body]="'This is a card body text.'"
    [buttonText]="'Learn More'"
    [buttonUrl]="'#'"
    [ariaLabelledby]="'card-title-Card Title'"
    [ariaDescribedby]="'card-body-This is a card body text.'"
    role="region"
  ></ds-card>

Buenas Prácticas

Para asegurar la accesibilidad, siga estas buenas prácticas:

  • Proporcione etiquetas accesibles para todos los elementos interactivos.
  • Use roles y atributos ARIA adecuados para describir la funcionalidad de los elementos.
  • Verifique la accesibilidad utilizando herramientas como el inspector de accesibilidad del navegador.
