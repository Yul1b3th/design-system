Inputs

Los componentes de entrada (ds-input) se utilizan para permitir a los usuarios ingresar datos en la aplicación. Estos componentes son altamente personalizables y soportan múltiples tipos de entradas para adaptarse a diferentes contextos de uso.

Tipos de Inputs

A continuación se presentan los diferentes tipos de componentes de entrada disponibles:

  • Text Input
  • Number Input
  • Email Input
  • Password Input
  • Date Input
  • Select Input
  • Checkbox
  • Radio
  • Textarea

States

Los diferentes estados del componente de entrada de texto muestran cómo se comporta el componente en diversas situaciones.

Default

Hover

El estado hover se muestra cuando el usuario pasa el cursor sobre el campo de entrada.

Typing (Focus)

El estado de enfoque se muestra cuando el usuario hace clic en el campo de entrada y comienza a escribir.

Disabled

Verification

El estado de verificación se muestra cuando el campo de entrada ha sido verificado.

Error

El estado de error se muestra cuando hay un error en el campo de entrada.

Text Input

El componente de entrada de texto (ds-text-input) se utiliza para permitir a los usuarios ingresar texto. Este componente es altamente personalizable y soporta múltiples configuraciones.

Uso

Ejemplo de uso en HTML:

<form [formGroup]="form">
  <ds-text-input
    formControlName="textInput"
    id="name"
    label="Name"
    placeholder="Enter text"
    [ariaRequired]="true"
  ></ds-text-input>
</form>

Configuración

El componente de entrada de texto acepta las siguientes propiedades de entrada (@Input):

  • formControlName (string): El nombre del control de formulario al que está vinculado el componente.
  • placeholder (string): Texto que se mostrará cuando el campo esté vacío.
  • disabled (boolean): Indica si el campo de texto está deshabilitado. Valor por defecto: false.

Number Input

El componente de entrada numérica (ds-number-input) se utiliza para permitir a los usuarios ingresar números. Este componente es altamente personalizable y soporta múltiples configuraciones.

Uso

Ejemplo de uso en HTML:

<form [formGroup]="form">
  <ds-number-input
    formControlName="numberInput"
    id="age"
    label="Age"
    placeholder="Enter age"
    [ariaRequired]="true"
    [min]="0"
    [max]="100"
    [step]="1"
  ></ds-number-input>
</form>

Configuración

El componente de entrada numérica acepta las siguientes propiedades de entrada (@Input):

  • formControlName (string): El nombre del control de formulario al que está vinculado el componente.
  • placeholder (string): Texto que se mostrará cuando el campo esté vacío.
  • disabled (boolean): Indica si el campo numérico está deshabilitado. Valor por defecto: false.
  • min (number): El valor mínimo permitido.
  • max (number): El valor máximo permitido.
  • step (number): El incremento entre valores del rango.

Select Input

El componente de entrada de selección (ds-select-input) se utiliza para permitir a los usuarios seleccionar una opción de una lista desplegable. Este componente es altamente personalizable y soporta múltiples configuraciones.

Uso

Ejemplo de uso en HTML:

<ds-select-input [(ngModel)]="selectedOption" [options]="options"></ds-select-input>

Configuración

El componente de entrada de selección acepta las siguientes propiedades de entrada (@Input):

  • ngModel (any): La opción seleccionada.
  • options (Array<{ label: string, value: any }>): Lista de opciones que se mostrarán en el menú desplegable. Cada opción debe tener una etiqueta (label) y un valor (value).
  • placeholder (string): Texto que se mostrará cuando no se haya seleccionado ninguna opción.
  • disabled (boolean): Indica si el menú desplegable está deshabilitado. Valor por defecto: false.

Ejemplos de Entrada de Selección

Textarea

El componente de área de texto (ds-textarea) se utiliza para permitir a los usuarios ingresar texto en múltiples líneas. Este componente es altamente personalizable y soporta múltiples configuraciones.

Uso

Ejemplo de uso en HTML:

<ds-textarea [(ngModel)]="textValue"></ds-textarea>

Configuración

El componente de área de texto acepta las siguientes propiedades de entrada (@Input):

  • ngModel (string): El valor del texto ingresado.
  • placeholder (string): Texto que se mostrará cuando el campo esté vacío.
  • rows (number): El número de filas visibles en el área de texto.
  • disabled (boolean): Indica si el área de texto está deshabilitada. Valor por defecto: false.
